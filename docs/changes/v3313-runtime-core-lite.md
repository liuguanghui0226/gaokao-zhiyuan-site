# v3.313 浏览器首屏轻量核心

## 目标

降低全国高考志愿网站的首屏下载、解压和 JSON 解析负担，同时保留完整来源证据、31 省运行分片和全部推荐边界。

## 实现

- 新增 `knowledge-core-lite.json.gz`，作为浏览器首屏核心。
- 完整 `knowledge-core.json.gz` 保持发布和可下载状态，继续用于证据审计、回归验证和后续增量构建。
- 5117 条来源说明在轻量核心中保留来源 ID、标题、链接、质量等级、省份、年份、录取日程和普通专科待发布复核字段。
- 省级录取、计划和位次数据仍按考生省份加载；全国 868426 条记录、116656 条位次换算和 31 省分片均未删减。
- 运行清单新增 `coreLite` 哈希、字节数和 `runtimeProfile`，前端资源版本升级为 `3.313.0`。
- 公开导航移除内部“整合轮次”和“音频转写”，五个面向考生的页面改为点击后按需渲染。

## 结果

- 解压体积：15469285 字节降至 3001319 字节，下降 80.60%。
- 压缩下载：2344217 字节降至 477036 字节，下降 79.65%。
- 本地五轮读取基准中，平均解压从 18.40 ms 降至 3.66 ms，平均 JSON 解析从 60.96 ms 降至 10.89 ms；网络耗时不作为本地基准结论。
- 14 个专业门类点击链路、推荐表单、位次换算来源标题、录取日程和 5 个专科线待发布省份的安全降级保持不变。
- 首屏不再预先生成 121 张整合轮次卡、124 条音频处理项和隐藏的数据来源列表。

## 边界

- 轻量核心只是浏览器传输层，不是数据删减或来源降级。
- 完整来源中的原始文件清单、SHA、OCR 审计和证据边界仍保存在完整核心与 GitHub Release 中。
- 以后每次新增录取数据后必须重新运行轻量核心构建器；回归会拒绝模型版本、来源数量、哈希或体积门槛不一致的发布。

## 验收

1. 轻量核心与完整核心模型版本、结构化记录数、位次换算数和来源说明数一致。
2. 所有轻量来源字段都在显式白名单内，未泄露大体积原始证据字段。
3. 解压体积至少下降 75%，压缩体积至少下降 65%。
4. 公网清单同时校验完整核心与轻量核心的 SHA-256。
5. 当前发布回归共 59 项。
